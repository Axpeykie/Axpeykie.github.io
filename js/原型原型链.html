# 原型/原型链

var person1 = new Person();

以上是用构造函数创建一个实例化对象的过程。
ES中每个函数都有一个prototype的属性，表示函数的原型对象。
Person.prototype 表示构造函数Person的原型对象。
ES中每个对象都有一个__proto__属性，指向对象的原型对象。
person1.__proto__ 表示实例对象person1的原型对象，即Person.prototype

console.log(person1.__proto__ === Person.prototype);//true

原型对象也是对象，因此也有__proto__属性，指向它自己的原型对象。这里称为原型的原型。
如果某个对象不是由构造函数生成的，那么默认new Object() 生成。
所以Person.prototype.__proto__指向Object.prototype

console.log(Person.prototype.__proto__ === Object.prototype);//true

Object.prototype 的原型对象是null。此处是原型链的终点。

console.log(Object.prototype.__proto__);//null

当我们访问对象的一个属性时，首先会到对象本身的属性去找。
如果找不到，到对象的原型对象去找，再找不到就到原型的原型中找，直到Object.prototype(原型链结束)

constructor属性：
每个原型都有constructor属性，关联它的构造函数。

console.log(Person === Person.prototype.constructor); //true
console.log(Person === person1.__proto__.constructor); //true

# 关于继承

引用《你不知道的JavaScript》中的话，

继承意味着复制操作，然而 JavaScript 默认并不会复制对象的属性，相反，JavaScript 只是在两个对象之间创建一个关联，这样，一个对象就可以通过委托访问另一个对象的属性和函数，所以与其叫继承，委托的说法反而更准确些。

# 扩展

当我们将两个函数（暂称源函数与目标函数）的原型指向同一个地址时，原型上的属性将被共享，但修改任何一个函数的原型都会影响到另一个函数的原型。如果不想这样，如何解决？

  1. 创建一个空函数，将空函数的原型与源函数的原型指向同一个地址。这样空函数的原型也拥有了源函数原型上的属性。
  2. 以这个空函数作为构造函数生成一个实例对象。生成的实例对象将继承原型的所有属性。
  3. 将上一步生成的实例对象作为目标函数的原型。

这样，目标函数的原型就拥有源函数原型上的所有属性了，而且对它进行修改将不会影响源函数的原型。

function Person() {};//源函数

function cyPerson() {};//目标函数

function nfun() {};
nfun.prototype = Person.prototype;
cyPerson.prototype = new nfun();

测试：

Person.prototype.name = "Tom";
console.log(cyPerson.prototype.name); //"Tom"
cyPerson.prototype.name = "Rose";
console.log(Person.prototype.name); //"Tom"
